Python debugging

Common Python Errors and Fixes

Use this guide when YBY Compiler or Python shows an error message. Start with the first error line, check the line number, then compare the message with the explanations below.

SyntaxError

A SyntaxError means Python could not understand the structure of your code. Check for missing colons, brackets, commas, quotes, or parentheses.

if score >= 50
    print("Pass")

Fix it by adding the missing colon:

if score >= 50:
    print("Pass")

IndentationError

An IndentationError means spaces are not aligned correctly. Code inside loops, functions, and if-statements must be indented consistently.

for number in range(3):
print(number)

Fix it by indenting the body:

for number in range(3):
    print(number)

NameError

A NameError means a variable or function name was used before it was created, or the spelling does not match.

print(total)
total = 10

Fix it by defining the value before using it:

total = 10
print(total)

TypeError

A TypeError means incompatible data types were used together, such as adding text directly to a number.

age = 12
print("Age: " + age)

Fix it by converting the number to text or by using an f-string:

age = 12
print(f"Age: {age}")

ValueError

A ValueError means Python received the right type of value, but the value itself is not valid for the operation.

number = int("hello")

Fix it by validating input before converting:

text = "42"
if text.isdigit():
    number = int(text)
    print(number)

FileNotFoundError

A FileNotFoundError means Python cannot find the file path. In YBY Compiler, upload the file, click Attach to Python, then use the exact path shown under the upload tool.

import pandas as pd

df = pd.read_csv("/home/pyodide/data/my-file.csv")
print(df.head())

Debugging checklist

  • Read the first error line before reading the rest of the traceback.
  • Check the line number mentioned in the output console.
  • Look for spelling differences in variable names.
  • Check indentation after if, for, while, def, and class.
  • If your program reads a file, attach the file first and use the /home/pyodide/data/ path.
